About this app

Include information here about your app.

  • One Button Calculator switch is fixed to X
    • Press Space start. Blue box will loop through the columns.
    • Press Space to select desire column
    • Thick Green Box will loop around the rows. Press Space to select the required button.
    • Press Space to repeat the process
  • Test
    • To test One button switch, I designed 4 different test sets. Each set consisting of 5 different equations.
    • Each test set was set in different time interval i.e loop time of the selector box.
    • 4 different time set used were: 300ms, 700ms, 500ms, 1000ms
    • Total number of experiments = 4 * 5 = 20
  • Equations used
    • 1234 + 7890
    • ((2*5) + 10) - 2
    • ((1200/35) *101)
    • (5 *6) -29
    • ((410 * 2)/6)-813
    • Parenthesis are not used in the calculator. It just shows order of operation to the user
  • Results
    • While using 300 and 500 ms interval time, the user was able to complete each test slightly faster. However, there were far many errors which resulted longer completion time as user had to repeat the whole process from start.
    • Using 700 ms, the user was able to complete the given task with significantly less errors and repetation. But, few errors still exists
    • Using 1000 ms, the user was ablr to complete all task with no errors at all. It took slightly more time to complete the task but there was no repeatation of the task so overall time was still faster than the previous cases.
  • From above experiments, I selected 1000 ms to be interval time as it seems to be better and safer options in terms of error rate and overall completion time.
  • Anything else you have tried out
  • While doing the experiments, major errors led to repetation of the task because there was no backspace or single delete option available. So, I tired to implement deletion using double press of the key. However, I was not successful to implement it properly. As double press of key was consider as two single press.
